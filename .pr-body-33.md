## How It Works

### Registration
```bash
POST /api/v1/auth/register
{
  "email": "owner@roofing.com",
  "password": "securepass",
  "business_name": "Ace Roofing"
}
→ Creates Business + User, returns JWT token
```

### Login
```bash
POST /api/v1/auth/login
{
  "email": "owner@roofing.com",
  "password": "securepass"
}
→ Returns JWT token
```

### Protected Endpoints
All `/api/v1/calls/`, `/api/v1/businesses/me`, etc. now require:
```
Authorization: Bearer <token>
```

### Multi-Tenancy
- Each user belongs to one business (`user.business_id → business.id`)
- All queries automatically scope to the authenticated user's business
- Users can only see their own calls, business settings, etc.
